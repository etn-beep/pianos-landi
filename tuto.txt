GITHUB SITE:

Aller dans le dossier du projet
cd ~/mon-site-docs

Activer l'environnement virtuel (INDISPENSABLE)
source .venv/bin/activate 

---

git add .
git commit -m "comments"
git push origin main  

---

Publier le site en ligne (Pour vos visiteurs)
properdocs gh-deploy --force

---


OU: 
git add . && git commit -m "update" && git push && properdocs gh-deploy --force

Ajouter une nouvelle page (Exemple)
Si vous voulez créer une nouvelle section (ex: "Entretien") :

-Créer le fichier :

mkdir docs/entretien

nano docs/entretien/index.md   

(Écrivez votre contenu Markdown et sauvegardez).
Déclarer dans le menu : Ouvrez properdocs.yml et ajoutez la 
ligne dans la section nav: :
nav:
  - -# ... vos autres pages ...
  - Entretien: entretien/index.md
  

  
  
  - En cas de problème
Commande properdocs introuvable ? → Vous avez oublié l'étape 1 (source .venv/bin/activate).
Site non mis à jour sur GitHub ? → Vérifiez l'onglet Actions sur GitHub. Si c'est rouge, cliquez dessus pour voir l'erreur (souvent une faute de frappe dans properdocs.yml).
Images non affichées ? → Vérifiez le chemin (ex: ../images/photo.jpg si vous êtes dans un sous-dossier).
